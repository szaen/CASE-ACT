# Guia de Contribuição

Estamos empolgados que você quer contribuir com este projeto! Aqui estão algumas diretrizes para ajudar
você a colaborar de maneira eficiente.

## Como Contribuir

1. **Fork o repositório**: Crie uma cópia do repositório na sua conta do GitHub.
2. **Clone o repositório**: Clone o repositório para a sua máquina local.
   
   ```bash
   git clone https://github.com/seu-usuario/lançamentos-financeiros.git
   ```

3. **Crie um branch para sua contribuição**:
   
   ```bash
   git checkout -b minha-contribuicao
   ```

4. **Faça suas modificações**: Implemente suas alterações, como correções de bugs, novas funcionalidades ou
melhorias na documentação.

6. **Escreva testes**: Certifique-se de que o código que você escreveu está coberto por testes, se aplicável.

7. **Commit suas alterações**: Use mensagens de commit claras e descritivas.

   ```bash
   git commit -m "Descrição clara da alteração"
   ```

8. **Envie o branch para o seu fork no GitHub**:
   
   ```bash
   git push origin minha-contribuicao
   ```

9. **Abra um Pull Request**: Envie suas alterações para revisão abrindo um Pull Request (PR) para o repositório principal.

## Código de Conduta

- Seja respeitoso e profissional em suas comunicações.
- Foque em adicionar valor ao projeto e em compartilhar conhecimentos.
- Qualquer tipo de assédio, discriminação ou comportamento abusivo não será tolerado.

## Padrão de Código

Por favor, siga o padrão de código Java e utilize as convenções do [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html).

- **Indentação**: 2 espaços.
- **Nomes de variáveis e métodos**: utilize `camelCase`.
- **Nomes de classes**: utilize `PascalCase`.

## Requisitos para Pull Requests

- Cada PR deve ser focado em uma única alteração ou melhoria.
- Certifique-se de que todos os testes passam antes de enviar um PR.
- Inclua descrições detalhadas nas mensagens de commit e no próprio PR.
- **Documentação**: Atualize a documentação, se necessário, para refletir as mudanças feitas.

## Revisão de Código

Nosso processo de revisão de código é uma etapa importante para garantir a qualidade. Seja aberto ao feedback e, se for necessário, faça ajustes em seu PR.

## Configurações de Ambiente

Para configuração do ambiente, consulte o arquivo `README.md` na raiz do projeto.

## Feedback e Sugestões

Estamos sempre abertos a sugestões e melhorias. Se tiver uma ideia, abra uma *issue* para discutirmos!

Obrigado por contribuir para o projeto!
